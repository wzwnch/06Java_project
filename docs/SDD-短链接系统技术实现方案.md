        │
        ▼

## 四、接口/数据结构/字段设计

### 4.1 数据库表设计

#### 4.1.1 用户表（link_user）

| 字段名 | 类型 | 长度 | 必填 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | Y | 主键，雪花算法 |
| username | VARCHAR | 64 | Y | 用户名，唯一索引 |
| password | VARCHAR | 256 | Y | 密码（加密） |
| phone | VARCHAR | 256 | N | 手机号（加密） |
| mail | VARCHAR | 256 | N | 邮箱（加密） |
| real_phone | VARCHAR | 32 | N | 手机号（脱敏展示用） |
| real_mail | VARCHAR | 64 | N | 邮箱（脱敏展示用） |
| del_flag | TINYINT | - | Y | 删除标志：0-正常，1-删除 |
| create_time | DATETIME | - | Y | 创建时间 |
| update_time | DATETIME | - | Y | 更新时间 |

**分片策略**：按 username 哈希分片

#### 4.1.2 短链接分组表（link_group）

| 字段名 | 类型 | 长度 | 必填 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | Y | 主键，雪花算法 |
| gid | VARCHAR | 32 | Y | 分组唯一标识 |
| name | VARCHAR | 64 | Y | 分组名称 |
| username | VARCHAR | 64 | Y | 所属用户 |
| sort_order | INT | - | Y | 排序值，默认 0 |
| del_flag | TINYINT | - | Y | 删除标志 |
| create_time | DATETIME | - | Y | 创建时间 |
| update_time | DATETIME | - | Y | 更新时间 |

**索引**：`idx_username` (username)，`idx_gid` (gid)

#### 4.1.3 短链接表（link_link）

| 字段名 | 类型 | 长度 | 必填 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | Y | 主键，雪花算法 |
| short_code | VARCHAR | 16 | Y | 短链接码，唯一索引 |
| gid | VARCHAR | 32 | Y | 分组标识 |
| origin_url | VARCHAR | 2048 | Y | 原始 URL |
| favicon_url | VARCHAR | 512 | N | 网站图标 URL |
| title | VARCHAR | 256 | N | 网站标题 |
| expire_time | DATETIME | - | N | 过期时间，NULL 表示永不过期 |
| status | TINYINT | - | Y | 状态：0-正常，1-回收站 |
| del_flag | TINYINT | - | Y | 删除标志 |
| create_time | DATETIME | - | Y | 创建时间 |
| update_time | DATETIME | - | Y | 更新时间 |

**分片策略**：按 short_code 哈希分片
**索引**：`idx_gid` (gid)，`idx_status` (status)

#### 4.1.4 短链接访问日志表（link_link_stats）

| 字段名 | 类型 | 长度 | 必填 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | Y | 主键，雪花算法 |
| short_code | VARCHAR | 16 | Y | 短链接码 |
| gid | VARCHAR | 32 | Y | 分组标识 |
| pv | BIGINT | - | Y | PV 计数 |
| uv | VARCHAR | 64 | N | UV 标识（用户指纹） |
| uip | VARCHAR | 64 | N | UIP 标识（IP） |
| ip | VARCHAR | 64 | N | 访问 IP |
| region | VARCHAR | 64 | N | 地区 |
| os | VARCHAR | 32 | N | 操作系统 |
| browser | VARCHAR | 32 | N | 浏览器 |
| device | VARCHAR | 32 | N | 设备类型 |
| network | VARCHAR | 32 | N | 网络类型 |
| create_time | DATETIME | - | Y | 创建时间 |

**分片策略**：按 create_time 按月分表
**索引**：`idx_short_code` (short_code)，`idx_create_time` (create_time)

#### 4.1.5 短链接统计汇总表（link_link_stats_today）

| 字段名 | 类型 | 长度 | 必填 | 说明 |
|--------|------|------|------|------|
| id | BIGINT | - | Y | 主键 |
| short_code | VARCHAR | 16 | Y | 短链接码 |
| gid | VARCHAR | 32 | Y | 分组标识 |
| date | DATE | - | Y | 统计日期 |
| pv | BIGINT | - | Y | 当日 PV |
| uv | BIGINT | - | Y | 当日 UV |
| uip | BIGINT | - | Y | 当日 UIP |
| create_time | DATETIME | - | Y | 创建时间 |
| update_time | DATETIME | - | Y | 更新时间 |

**唯一索引**：`uk_short_code_date` (short_code, date)

### 4.2 Redis 缓存 Key 设计

| Key 模式 | 类型 | 过期时间 | 说明 |
|----------|------|----------|------|
| `link:info:{shortCode}` | String | 1 小时 | 短链接信息缓存 |
| `link:empty:{shortCode}` | String | 5 分钟 | 空值缓存（防穿透） |
| `link:user:token:{token}` | String | 24 小时 | 用户 Token 缓存 |
| `link:user:info:{username}` | Hash | 30 分钟 | 用户信息缓存 |
| `link:bloom:username` | Bloom | 永久 | 用户名布隆过滤器 |
| `link:bloom:shortcode` | Bloom | 永久 | 短链接码布隆过滤器 |
| `link:stats:pv:{shortCode}:{date}` | String | 7 天 | 当日 PV 计数 |
| `link:stats:uv:{shortCode}:{date}` | Set | 7 天 | 当日 UV 集合 |
| `link:stats:uip:{shortCode}:{date}` | Set | 7 天 | 当日 UIP 集合 |
| `link:lock:{shortCode}` | String | 10 秒 | 分布式锁 |
| `link:queue:log` | List | - | 访问日志队列 |

| 配置项 | 方案 |

### 5.2 缓存策略方案

#### 5.2.1 缓存穿透防护

| 场景 | 方案 |
|------|------|
| 查询不存在的用户名 | 布隆过滤器 + 空值缓存（5 分钟） |
| 查询不存在的短链接 | 布隆过滤器 + 空值缓存（5 分钟） |

#### 5.2.2 缓存击穿防护

| 场景 | 方案 |
|------|------|
| 热点短链接过期 | Redisson 分布式锁，单线程重建缓存 |
| 锁超时处理 | 设置锁超时时间 10 秒，防止死锁 |

#### 5.2.3 缓存预热方案

| 场景 | 方案 |
|------|------|
| 系统启动 | 定时任务预热 Top 1000 热门短链接 |
| 新建短链接 | 创建时直接写入缓存 |

#### 5.2.4 缓存更新策略

| 场景 | 方案 |
|------|------|
| 短链接修改 | 先更新数据库，再删除缓存 |
| 短链接删除 | 先更新数据库，再删除缓存 |
| 用户信息修改 | 先更新数据库，再删除缓存 |

## 六、潜在风险与规避方案

### 6.1 性能风险

| 风险 | 影响 | 规避方案 |
|------|------|----------|
| 短链接跳转高并发 | 响应延迟、服务不可用 | 多级缓存、布隆过滤器、分布式锁、限流降级 |
| 统计数据写入压力 | 数据库瓶颈 | Redis 队列异步写入、批量合并 |
| 分库分表跨片查询 | 查询性能下降 | 冗余存储、ES 同步查询 |

### 6.2 数据风险

| 风险 | 影响 | 规避方案 |
|------|------|----------|
| 短链接码冲突 | 数据覆盖 | 雪花算法 + Base62 编码，冲突检测重试 |
| 分库分表数据倾斜 | 单点压力 | 分片键选择均匀分布字段 |
| 缓存与数据库不一致 | 数据错误 | 延迟双删、消息队列最终一致性 |

### 6.3 安全风险

| 风险 | 影响 | 规避方案 |
|------|------|----------|
| 恶意短链接（钓鱼） | 平台责任风险 | 目标 URL 安全检测、黑名单机制 |
| 暴力破解短链接码 | 隐私泄露 | 短链接码长度 ≥ 6 位、访问频率限制 |
| SQL 注入 | 数据泄露 | 预编译 SQL、参数校验 |
| XSS 攻击 | 用户信息泄露 | 输出转义、CSP 策略 |

### 6.4 可用性风险

| 风险 | 影响 | 规避方案 |
|------|------|----------|
| Redis 宕机 | 缓存失效、队列丢失 | 主从复制、哨兵模式、降级直连数据库 |
| 数据库宕机 | 服务不可用 | 主从复制、读写分离、自动故障转移 |
| 分库分表单点故障 | 部分数据不可用 | 每个分库配置主从 |

### 6.5 业务风险

| 风险 | 影响 | 规避方案 |
|------|------|----------|
| 短链接被滥用 | 资源消耗、合规风险 | 用户创建数量限制、有效期限制 |
| 热点短链接流量激增 | 缓存击穿、服务压力 | 提前预热、限流保护 |
| 访问日志数据膨胀 | 存储成本增加 | 定期归档、冷热分离 |
